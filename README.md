# Oasis

**A calm home base for people who build with AI.**

Oasis is a small, private desktop dashboard for people who work with AI coding
tools — built for **Claude Code** and **Codex** on **macOS and Windows**. The whole interface
floats as glass over a living sea: ask anything, catch an idea and develop it
into angles, keep your day's tasks and a journal, browse the images you generate,
launch your projects, and score the work with lo-fi, ambient, Spotify, or YouTube.

It runs as a tiny, near-zero-dependency Node app on your own machine (one
optional package powers the built-in terminal). **Free** — no account, no
telemetry, nothing phoned home.

---

## What's inside

- **The water** — a looping cinemagraph of open sea as your backdrop. Four moods
  (dawn / day / dusk / night) follow your clock, or pin one. Dark by design, so
  text always reads.
- **Ask** — the hero search bar runs through your own local `claude` CLI. Ask a
  question, work a problem, or capture a thought — then turn any answer into a
  task, an idea, or a journal note.
- **Voice — talk to Oasis.** Opt-in, hands-free or push-to-talk. Set a wake word
  ("Oasis…", or make it "Jarvis"), or just tap the floating orb, and speak: *"add
  a task… / remember… / brief me / start a 25 minute timer / make it night /
  launch <project>"* — and anything else becomes an **Ask**, answered by your local
  Claude and **read back aloud**. Built on the browser's own speech engine (no
  extra install). The mic only opens while you're listening, and it's off until you
  switch it on.
- **Ideas** — catch an idea in a line, then **develop** it into four angles —
  MVP cut, wild version, money angle, and first build step — generated by your
  local Claude. Pin the keepers; turn any angle straight into a task.
- **Relay** — put **Claude and Codex on the same task** so the pair beats either
  alone. **Delegate** has Claude architect a plan, Codex build on and critique it,
  Claude refine, then a clean synthesis; **Debate** has them argue opposing sides
  to a neutral verdict. Each turn sees the other's, and is told to push back when
  the partner is wrong, so mistakes get caught. Pick 1–3 rounds, watch the turns
  stream in, then copy the result, keep it as an idea, or hand it to a live Claude
  terminal. (Codex runs read-only; no `codex` CLI installed? Claude runs both
  sides and Oasis tells you.)
- **Today** — a drag-to-reorder task list; finished items sink to the bottom. Hand
  any task to **Claude** — it opens a terminal in your project, with the task
  staged and ready to run.
- **Journal** — a few lines and a mood whenever you need to think out loud, with
  a seven-day mood pulse.
- **Gallery** — an auto-updating wall of every image you generate (it watches
  your Codex generated-images folder and Oasis's own assets), with a lightbox,
  copy-path, reveal-in-folder, and paste-a-URL-to-keep.
- **Daily briefing** — once a day, one warm, grounding sentence about how your
  day is going, drawn from your tasks, journal and mood.
- **Music** — Lo-Fi, Ambient/downtempo, **Spotify**, or **YouTube**. Paste any
  Spotify or YouTube link, tap a preset, connect your own Spotify, or save your
  go-to YouTube playlists and mixes to "Your YouTube".
- **Tool dock** — auto-discovers the projects in your build folder (anything
  with a `Launch` script — `.bat` on Windows, `.command`/`.sh` on macOS — or an
  `npm run dev`) for one-click launch.
- **Recent agent work** — a quiet ticker of your latest Claude Code & Codex
  sessions; click to see them all, turn any into a task or journal note, or
  **resume one in a terminal**.
- **Terminals** — a movable, resizable terminal panel with a tab per session. Run
  a real, interactive Claude, Codex, or shell right inside Oasis: it opens as a
  small, frosted-glass window in the bottom-right, then drag it anywhere and
  resize it from the sides or corners (it remembers where you left it). Fresh
  terminals open **inside your Oasis project**; resume a past Claude session in
  its own project folder.
- **Edit anything inline** — double-click a task or a journal entry, or hit the
  edit (pencil) button on an idea, to rename or rewrite it right in place.
- **One-click backup** — export everything in your data folder to a single JSON
  file (Command palette → "Back up my data", or the **Back up data** button in
  Preferences). It stays on your machine.
- **In-app updates & restart** — when there's a new version, **Update now**
  applies it, then a **Restart now** button reloads Oasis on the new version — no
  manual relaunch needed.
- **Command palette** (`Ctrl K`), **customizable keyboard shortcuts** (press `?`
  to see them all and rebind any of them), **focus timer**, and **scenes** round it
  out.

## Requirements

- macOS (Apple silicon or Intel) **or** Windows 10 / 11
- [Node.js](https://nodejs.org/en/download) (free; LTS recommended)
- Recommended for the full experience: **Claude Code** — the local `claude` CLI
  powers Ask, Ideas "develop into angles", the daily briefing, and the
  Claude × Codex relay. **Codex** is optional: it powers the relay's second model,
  and its generated images flow into the Gallery. Oasis still runs fine without
  them — those panels simply stay quiet.

## Setup

**macOS — one line, recommended.** Open Terminal and paste:

```bash
curl -fsSL https://dakota1450.github.io/OASIS/install.sh | bash
```

This downloads Oasis, drops a Desktop launcher, and opens it. Installing from
Terminal sidesteps macOS Gatekeeper entirely — a browser-downloaded zip gets a
*quarantine* flag and recent macOS (Sequoia and later) blocks unsigned apps with
**no “Open Anyway” button**, so the double-click launchers can't start. Files
fetched by `curl` aren't quarantined, so there's nothing to approve. (Need Node
first? `brew install node` or [nodejs.org](https://nodejs.org/en/download).)

**Windows — download and run.** Grab `Oasis-Windows.zip` from the site, unzip
anywhere, and double-click **`Setup Oasis.bat`**.

**Downloaded the macOS zip anyway?** Clear the quarantine and make the launchers
runnable, once, in Terminal (drag the unzipped folder onto the Terminal window so
it's the current directory first):

```bash
xattr -dr com.apple.quarantine "$(pwd)" ; chmod +x *.command
```

…then double-click **`Setup Oasis.command`**. (The one-line installer above does
all of this for you.)

Either setup checks for Node.js, creates a launcher, and opens Oasis.
3. Oasis opens in its own window and runs a one-minute setup wizard: your name,
   your projects folder, **connecting Claude & Codex**, your default station, and
   your scene. That's it — it builds itself around your preferences.

   The **Connect Claude & Codex** step checks whether the `claude` and `codex`
   CLIs are installed and shows you, live, whether Ask / Ideas / the briefing and
   the relay are ready. If one isn't found it gives you the one-line install

   API keys to paste — Oasis just calls the CLI you're already signed into, and
   nothing about your plan leaves the machine. The step is optional; the rest of

After that, open Oasis any time from the **Oasis** shortcut (or
`Launch Oasis.bat` on Windows / `Launch Oasis.command` on macOS). To change
preferences later, click the settings icon in the top-right corner.

> **After an update**, run **`Restart Oasis`** (`.bat` / `.command`) rather than
> Launch. Launch reuses an already-running server, so it would keep showing the
> old version; Restart fully stops the server and reloads the new one.

> Oasis listens on `http://localhost:7777`. Everything stays on your machine.

### Optional: connect your own Spotify

Oasis plays Spotify through the official embed with no setup. To browse *your
own* playlists, open Music -> Spotify -> Connect, create a free app at the
[Spotify dashboard](https://developer.spotify.com/dashboard), add the redirect
URI shown in Oasis, and paste your Client ID. Uses PKCE — no client secret,
nothing leaves your machine except the standard Spotify login.

### YouTube

Open Music -> YouTube and paste any video, playlist, or mix link (or a `youtu.be`
/ `music.youtube.com` link) to play its audio through YouTube's privacy-mode
(`youtube-nocookie.com`) player. Tap **Save current** to keep your own playlists
and mixes under **Your YouTube** — they persist locally, with no Google sign-in
or API key to set up.

## Distributing Oasis (free download)

Oasis is free. To build a clean copy to hand out:

1. Run **`Package Oasis.bat`** (or `powershell -File package.ps1`).
2. It produces two clean zips — the app with **empty data and no personal
   files** — `dist\Oasis-Windows.zip` and `dist\Oasis-macOS.zip`, and copies both
   into `docs\download\`, which is what the marketing page links to (the page
   highlights the right one for each visitor's OS).
3. Commit and push `docs\` and enable **GitHub Pages** (see `PUBLISH.md`). The
   download buttons on the site then serve your zips directly.

The **marketing page** lives in `docs/` as a self-contained `index.html` (the
GitHub Pages site). You can also preview it locally at
`http://localhost:7777/site/`.

## Developer & agent docs

Building on Oasis, or pointing an AI agent at it? Start here:

- **[AGENTS.md](AGENTS.md)** — the single source of truth for AI coding agents
  (Codex reads it automatically; [CLAUDE.md](CLAUDE.md) points to it). Run
  commands, the hard constraints, conventions, and security rules.
- **[ARCHITECTURE.md](ARCHITECTURE.md)** — the request lifecycle, data model,
  every endpoint, the `claude` CLI integration, and the asset security model.
- **[CONTRIBUTING.md](CONTRIBUTING.md)** — local setup, coding conventions, how to
  add an endpoint, and how to build/publish.
- **[SECURITY.md](SECURITY.md)** — what's guarded and why; local-first, no
  telemetry.

Quick start for hacking: there's no build step — just `node server.js` and open
`http://localhost:7777`. The only `npm install` is the optional `node-pty` for the
embedded terminal (see Notes); everything else runs on Node's standard library.

## Notes

- Built to be light: the looping sea video caps its work and sleeps whenever the
  window is hidden, falling back to a still photo if a video can't play.
- The ocean backdrops were generated and recompressed for the web.
- **Almost** no npm dependencies — exactly one, and it's optional. The embedded
  **terminal** uses `node-pty`; run `npm install` once inside the Oasis folder to
  enable it. Without it, everything else works and the terminal button simply
  tells you it needs that one-time install. Nothing else leaves Node's standard
  library, and the frontend has no build step.
